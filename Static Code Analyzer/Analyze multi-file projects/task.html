Description

As a rule, real projects contain more than a single file. Also, project directories often contain not only Python files. We don't need to check if the HTML file follows PEP8.

We recommend to check out a tutorial on realpython.com that can help you figure out files and directories.

Objectives

In this stage, you need to improve your program so that it can analyze all Python files inside a specified directory.

Please note that:

  1. You also need to change the input format. Instead of reading the path from the standard input, the program must obtain it as a command-line argument.

    > python code_analyzer.py directory-or-file

  2. The output format also needs to be slightly changed. It should include the analyzed path:

    Path: Line X: Code Message

  3. All output lines must be sorted according to the file name (lexicographically), line number, and issue code.

  4. Other files must be skipped.

As before:

  • It is important that all the checks implemented in the previous stages still work properly.

  • If a line contains the same stylistic issue several times, your program must print the information only once. If a line has different issues with different codes, they should be printed in the sorted order.

  • To simplify the solution, we consider it acceptable if your program finds some false-positive stylistic issues in strings, especially multi-lines ('''...''' and """...""").

  • We recommend that you break your program code into a set of functions and classes to avoid confusion.

Examples

  1. Only a single file is specified as the input:

    > python code_analyzer.py /path/to/file/script.py

/path/to/file/script.py: Line 3: S003 Unnecessary semicolon
/path/to/file/script.py: Line 6: S001 Too long line
/path/to/file/script.py: Line 11: S006 More than two blank lines used before this line

/path/to/file/script.py: Line 13: S004 At least two spaces before inline comment required
/path/to/file/script.py: Line 13: S005 TODO found
