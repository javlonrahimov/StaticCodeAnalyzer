Description

One way to write beautiful and consistently formatted code with Python is following the PEP8 specification and practices recommended by the Python community. This is not always easy, especially for beginners. Luckily, there are special tools called static code analyzers (pylint, flake8, and others) that can automatically verify that your code meets all the standards and recommendations. The tools analyze your code and output all the issues they find. They don't execute your code, they only read it.

In this project, you will create a small static analyzer tool that finds some common stylistic mistakes in Python code. This way, you will get the general idea of static code analysis and improve your Python skills.

PEP8 says that we should "limit all lines to a maximum of 79 characters". For example, it keeps your code more readable. So, at the first iteration, we'll make a program that checks that lines in code are not too long.

Objectives

In this stage, your program must read Python code from a specified file and perform only a single check: lines in the code must not be longer than 79 characters.

Note that:

  1. The path to the file is obtained from the standard input.
  2. The general output format is: 
    Line X: Code Message

    In the format:

    • X is the number of the line where the issue was found. The count starts with one.

    • Code is the code of the particular stylistic issue (like S001).

    • Message is a human-readable description of the issue (optional).

    For example:

    Line 3: S001 Too long

    This format will be used throughout the project with some minor changes.

  3. The order of the lines must always be first to last.
  4. Your program can output another message instead of Too long. The rest of the output must exactly match the provided example. In this code S001, S means a stylistic issue, and 001 is the internal number of the issue.

Examples

Here is an example of the file's content:

print('What\'s your name?')
name = input()
print(f'Hello, {name}')  # here is an obvious comment: this prints greeting with a name

very_big_number = 11_000_000_000_000_000_000_000_000_000_000_000_000_000_000_000
print(very_big_number)

This code contains two long lines (>79 characters): lines 3 and 5.

Here is the expected output for the given example:

Line 3: S001 Too long
Line 5: S001 Too long
